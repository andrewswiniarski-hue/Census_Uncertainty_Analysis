# /data — never committed, always regenerated

Both subfolders are gitignored (only these placeholder files are tracked):

| Folder | Contents |
|--------|----------|
| `raw/` | Data exactly as pulled from Census APIs / downloads — never edited by hand |
| `processed/` | Cleaned or derived datasets built from `raw/` |

**If you just cloned this repo and these folders are empty, that's expected.**
Every file here is regenerated by the scripts in [`/ingestion`](../ingestion) — see each
script's docstring for what it produces. Keeping data out of git keeps the repo small
and guarantees the pipeline stays reproducible (a graded deliverable).
